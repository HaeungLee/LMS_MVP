# LMS 시스템 현황 분석 (2025-08-27)

## 1. 프론트엔드에 구현되지 않은 백엔드 기능

백엔드에 구현된 기능 중 현재 프론트엔드에서 사용되지 않아 사용자에게 제공되지 않는 기능 목록입니다.

### 주요 미구현 기능:

- **관리자 페이지 (`admin.py`):**
  - 사용자 관리, 콘텐츠(Taxonomy) 생성 등 관리자 전용 기능 전체
- **AI 기반 기능 확장 (`ai_features.py`, `personalization.py`):**
  - AI 자동 문제 생성 및 답변 분석 기능 (현재 테스트용으로만 존재)
  - 학생 맞춤형 학습 경로 추천 기능
- **사용자 피드백 및 프로필 (`feedback.py`, `student.py`, `beta_testing.py`):**
  - 학생 프로필 관리 기능
  - 시스템 피드백 제출 기능
  - 베타 테스트 관련 기능
- **모니터링 (`monitoring.py`):**
  - 백엔드 시스템 상태 확인 기능 (관리자 UI 부재)

---

## 2. 백엔드 코드 리팩토링 제안

코드의 유지보수성, 확장성, 안정성을 높이기 위한 개선 제안입니다.

### 주요 리팩토링 영역:

- **AI 관련 기능 통합:**
  - `ai_learning.py`, `ai_features.py` 등 여러 파일에 분산된 AI 관련 API를 기능에 따라 재구성 및 통합 필요.
- **API 역할 명확화:**
  - `submit.py`와 `ai_learning.py`의 중복될 수 있는 '제출' 기능 역할을 명확히 구분하고 코드 통합 고려.
- **설정 중앙 관리:**
  - `config.py`를 활용하여 흩어져 있는 설정 값(DB 접속 정보, API 키 등)을 중앙에서 관리하여 일관성 및 안정성 확보.
- **서비스 로직 분리:**
  - API 엔드포인트(라우터)의 비즈니스 로직을 `services` 디렉토리로 완전히 분리하여 코드 구조 개선.
- **일관된 오류 처리:**
  - FastAPI의 `Exception Handler`를 도입하여 API 전체의 오류 처리 방식과 응답 형식을 통일.
