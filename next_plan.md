## 🛠 기술적 구현 전략

### 데이터베이스 설계
```sql
-- 사용자 테이블
users (id, username, email, password_hash, role, created_at)

-- 문제 테이블  
questions (id, subject, topic, question_type, code_snippet, answer, difficulty, rubric, created_by, created_at)

-- 퀴즈 세션 테이블
quiz_sessions (id, user_id, started_at, completed_at, settings)

-- 답안 제출 테이블
submissions (id, session_id, question_id, user_answer, is_correct, skipped, submitted_at)

-- 피드백 테이블
feedbacks (id, submission_id, feedback_text, ai_generated, created_at)
```

### API 구조 확장
```
/auth
  POST /login
  POST /register
  POST /logout
  GET  /profile

/admin
  GET    /users
  GET    /questions
  POST   /questions
  PUT    /questions/:id
  DELETE /questions/:id

/analytics
  GET /dashboard
  GET /user-stats/:id
  GET /question-stats/:id
```

### 프론트엔드 구조 개선
```
src/
  components/
    auth/          # 로그인/회원가입
    admin/         # 교사용 관리 도구
    analytics/     # 통계 및 분석
    quiz/          # 기존 퀴즈 컴포넌트
  pages/
    AuthPage.jsx
    AdminPage.jsx
    AnalyticsPage.jsx
  hooks/
    useAuth.js
    useQuiz.js
    useAnalytics.js
```
