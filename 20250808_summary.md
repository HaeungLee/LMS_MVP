### 오늘 작업 요약 (Phase 0 + Phase 1)

## Phase 0 (계약/흐름 고정)
- `/submit` 요청/응답 계약 단일화 및 `submission_id` 반환
- 제출 성공 시 `/results/:submission_id`로 라우팅, 백엔드 조회 API 신설
- 결과 페이지에 `summary`, `recommendations` 표시, 네비게이션의 직접 결과 링크 제거
- 대시보드 `username` 의존 제거, 퀴즈 타이머 카운트다운/만료 처리 반영

## Phase 1 (DB 영속화 + 최소 시각화)
- ORM 스키마 추가: `submissions`, `submission_items`, `feedbacks`, `questions`
- Alembic 초기화 및 마이그레이션 생성/적용 완료
- `/submit` 시 DB 영속화(총점/문항별, `time_taken`, `summary`, `recommendations` 저장)
- `/results/{submission_id}` DB 우선 조회(메모리 폴백)
- `backend/data/db.json`에서 질문 30문항 시드 완료
- `/dashboard/stats`를 DB 집계 기반으로 전환: `topics`/`difficulties`/`recent_activity`/`topic_accuracy`
- 인덱스 추가: `submission_items.topic`, `submissions.submitted_at`
- 30초 메모리 캐시 도입, `subject` 필터 지원
- 프론트 차트 어댑터 도입 + Victory 연결: 주제별 문제 수(bar), 주제별 정답률(donut) 적용
- 결과 페이지에 `submitted_at` 표기 추가, 대시보드에 `topic_accuracy` 섹션 표시

## 비고/다음 조치
- 프론트에서 차트 렌더를 위해 패키지 설치 필요: `cd frontend && npm install && npm run dev`
- 대시보드에 과목 선택 UI 추가(백엔드 `subject` 필터와 연동)
- 서버 에러 로깅/모니터링 보강, 요약/추천의 DB 단일 소스화 검증
